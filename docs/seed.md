# 資料填充指令 (Seed Command)

為了方便開發與測試，本專案提供了一個 Django 管理指令，用於快速生成合理的假資料。

## 快速開始

執行此指令前，請確保已安裝開發與測試相關的相依套件：

```bash
pip install -e ".[dev,test]"
```

執行填充指令：

```bash
python manage.py seed
```

## 參數說明

| 參數 | 說明 | 預設值 |
|------|------|--------|
| `--amount` | 生成資料的數量級別 (`small`, `medium`, `large`) | `small` |
| `--clear` | 執行前是否清空現有資料（不包含 Superuser） | `True` |
| `--no-clear` | 執行前不清空資料 | - |

### 數量級別對照表

| 級別 | 用戶數 | 官方書目 | 分享書籍 |
|------|--------|----------|----------|
| `small` | 10 | 20 | 30 |
| `medium` | 30 | 50 | 100 |
| `large` | 100 | 200 | 500 |

## 帳號資訊

生成的用戶帳號具有以下特徵：

- **使用者名稱**：`user0`, `user1`, `user2` ... 依此類推。
- **密碼**：統一為 `testpass123`。
- **權限**：均為一般會員（`is_superuser=False`）。

> **注意**：指令執行時會保留所有 `is_superuser=True` 的管理員帳號，不會被 `--clear` 刪除。

## 媒體檔案處理

為了最小化磁碟空間佔用並確保 Django `ImageField` 運作正常：

1. 指令會自動在 `media/` 目錄下產生一個 1x1 像素的合法 JPG 檔案 `dummy.jpg`。
2. 所有生成的書籍照片與頭像均會引用此檔案。
3. 這樣可以避免 0-byte 檔案導致的讀取錯誤，同時節省開發環境的空間。

## 實作細節

- **生成器**：使用 `factory_boy` 搭配 `tests/factories.py` 定義的 Factory 類別。
- **業務邏輯**：
    - 自動為 `OCCUPIED` (借閱中) 狀態的書籍建立對應的 `Deal` (交易) 記錄。
    - 自動更新書籍的 `keeper` (持有者) 以符合面交後的狀態。
    - 使用 `transaction.atomic()` 確保資料生成的一致性。
