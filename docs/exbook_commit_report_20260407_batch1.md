## 功能與修改摘要

### 1. 核心邏輯：用 `django-fsm` 重構狀態轉換 (關鍵)
以前的狀態轉換程式碼散落在各處，很難追蹤且容易出錯。我們現在全面導入了狀態機模式：
- **交易 (Deal)**：重構了受理、拒絕、面交到完結的所有流程，並用 Signal 處理副作用。
- **書籍 (SharedBook)**：統一了上架、下架、借出、逾期等狀態。
- **借閱延長 & 申訴 (LoanExtension, Appeal)**：同步重構其審核與處理流程。
- **效益**：現在狀態轉換有 `@transition` 保護，無法直接改 `status` 欄位（`protected=True`），這讓系統變得很穩，不會出現奇怪的異常狀態。

### 2. 前端最佳化：Tailwind CSS 與 Crispy Forms
- **樣板重構**：拿掉舊的原生 CSS，改用 Tailwind 實作，載入速度跟維護性都好很多。
- **表單遷移**：將 6 個主要表單（像是借閱申請和評價）改用 Crispy Forms，程式碼乾淨不少，介面也統一了。
- **細節修補**：修復了 31 個樣板中超過 475 處的 CSS 類別合併錯誤，並重新整理了全站排版。

### 3. 手機版 UX 提升 (PWA 優化)
- **手勢支援**：加上了下拉重新載入 (Pull-to-refresh) 功能與對應的指示器。
- **導航一致性**：廢除 Sidebar，改以 Bottom Nav 為主，並針對劉海螢幕做了 Safe Area 處理。
- **載入預覽**：為書籍、交易和訊息列表加了 Skeleton Screens（骨架屏），滑起來感覺更流暢。
- **鍵盤處理**：導入 `visualViewport` API 解決手機虛擬鍵盤擋住輸入框的老問題。

### 4. 功能開發
- **套書管理**：前台介面補齊，現在可以在「我的書架」直接管理套書了。
- **違規機制**：新增了處分管理功能，包含警告、暫時停權和永久停權的處理邏輯。
- **實拍照片**：面交後（MEETED 狀態）可以直接上傳書況照片，並記錄在書籍的時間軸上。
- **Web Push**：完成通知發送服務的單元測試與 Mock 整合。

### 5. 效能與技術債清理
- **資料庫索引**：針對常用查詢（WishListItem, Deal, Notification 等）加了索引。
- **N+1 問題**：修復了願望書車的查詢邏輯，減少不必要的資料庫壓力。
- **風格修正**：用 `ruff` 修正了 27 個老舊的排版與語法錯誤。
